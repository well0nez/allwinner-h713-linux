## Session 2026-04-01 (continued) — Fan Tachometer IRQ Investigation Final

### External worker investigation results:
- Replaced hrtimer polling with devm_request_irq() + gpio_to_irq() path
- Added debug instrumentation (irq_total, pulse_count, sampled_level_toggles)
- Added no_rpm_shutdown module parameter for safe debugging
- Removed pwm0_pins DTS conflict on PH17

### Worker findings confirmed our analysis:
- IRQ registers correctly (sunxi_pio_edge)
- GPIO value toggles (fan pulses visible)
- IRQ counter stays at 0 — EINT hardware never fires
- irq_total=0 across all tests despite sampled_level_toggles incrementing
- IRQ was registered on wrong GPIO (209 instead of 241) — bank mapping issue

### Decision: REVERT to hrtimer polling
- GPIO-IRQ on H713 extended PH pins (PH11-PH19) is not viable
- EINT hardware does not route interrupts for these pins
- hrtimer polling at 100us is confirmed working and stable (2535-2565 RPM)
- This matches the only approach that delivers correct readings on this SoC
- Reverted all worker changes, restored IIO consumer + hrtimer polling version
- Fan RPM verified stable after revert

### Root cause (best understanding):
The H713 SoC has extended PH pins (PH11-PH19) that are NOT connected to the
EINT interrupt controller at the hardware level. The pinctrl driver can register
IRQ mappings (sunxi_pio_edge) but the physical interrupt line does not exist.
This is a silicon limitation, not a software bug. The stock kernel may use a
vendor-specific EINT initialization that we don't have access to, or the stock
may also use polling for these pins (needs SDK to confirm).
